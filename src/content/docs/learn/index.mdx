Most of this site is a reference: pages you look up when you need them. This is the other thing. **Learn the 80x Method** is a short course you read in order, start to finish, the way you would work through [React Foundations](https://nextjs.org/learn/react-foundations) before building with a new framework. By the end you will be able to look at any "AI for your fund" pitch, or any system you want built, and place it inside a method you understand.

## What "the 80x Method" means

80x is not a product you install. It is a **method**: a small set of patterns for building and running software inside a venture fund, using AI [agents](/reference/agents/) and AI coding assistants, that one person can ship and operate. The open-source tools on this site are just reference implementations of the method. The method itself is five patterns:

1. **Your CRM is the database.** One system is the fund's single source of truth. Every automation, dashboard, and agent reads and writes there, and nowhere else.
2. **Agents do the work.** An agent is an AI model run in a loop with tools, a goal, and a stopping rule. That loop is the unit of work behind meeting prep, screening, and pipeline hygiene.
3. **Read-only before write.** The first version of any agent can only look, never change. Safety comes from the tools you withhold, not from a promise to behave.
4. **Run it on a schedule.** Most fund software is not a live app. It is a small job that wakes on a timer, reconciles the truth, and goes back to sleep, with no server to maintain.
5. **Spec-first, and verify.** Write down what the system should do before building it, preview every change before it is real, and check the output against the source.

Everything else on this site is one of these five patterns applied to a specific fund workflow.

## Who this course is for

Partners, principals, and operations leads who want to understand and direct this class of software, whether you build it yourself or hire it built. No engineering background is assumed. Every term is defined in plain English the first time it appears, and no chapter expects you to have opened a terminal. If you already write code, you lose nothing but a few definitions you already knew.
